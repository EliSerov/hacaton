# План дальнейшего развития

## Качество поиска
- Reranker (cross-encoder) для улучшения релевантности результатов.
- Гибридный поиск (dense + sparse) и/или BM25.
- Дедупликация по контенту и качественная нормализация авторов/тем.

## Масштабирование
- Горизонтальное масштабирование RAG:
  - отдельный embedding-сервис (CPU),
  - отдельный LLM-сервис (GPU),
  - очереди приоритизации (search vs quiz).
- Кэширование результатов (Redis).

## Расширение источников и форматов
- Поддержка PDF/DOCX/HTML: пайплайн извлечения текста → чанкинг → индексация.
- Автоматический планировщик переиндексации.
- Мультиязычность (RU+EN) с выбором модели эмбеддингов и LLM.

## Безопасность
- JWT/подпись сообщений между сервисами вместо общего API key.
- Role-based access (admin/user), аудит действий.
- Sanitization логов (редактирование PII/секретов).

## UX
- Inline-кнопки для фильтров и быстрых “follow-up” вопросов.
- Сохранение “истории поиска” и избранного.
